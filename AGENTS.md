# AGENTS.md

## Commands

```bash
npm run dev       # dev server at http://localhost:4321
npm run build     # production build (also runs tsc implicitly)
npm run preview   # preview production build locally
npx tsc --noEmit  # manual typecheck without building
```

No lint, format, or test scripts exist. There is no test suite.

## Architecture

Single Astro app (not a monorepo). One page: `src/pages/index.astro` mounts `<PolisherApp client:load />`.

- `src/components/PolisherApp.tsx` — all app state; the only React island
- `src/components/SetupModal.tsx` — blocking modal for API key + provider setup
- `src/lib/providers/` — provider abstraction; add new LLM providers here
- `src/pages/api/polish.ts` — server-side CORS proxy to Abacus API (Vercel serverless function)
- `src/styles/global.css` — CSS custom properties and resets
- `src/components/PolisherApp.css` — all component-level styles

## Hybrid rendering quirk

`astro.config.mjs` does **not** set `output: "hybrid"`. Hybrid mode is activated implicitly because `src/pages/api/polish.ts` exports `export const prerender = false`. Any new API routes also need this export.

## API proxy

The frontend never calls the Abacus API directly. It always calls `POST /api/polish` (same-origin). The proxy forwards to `https://routellm.abacus.ai/v1/chat/completions`. No `.env` file and no `import.meta.env` references anywhere — the API key flows through the request body at runtime.

## localStorage keys

- `polisher.apikey` — user's API key
- `polisher.provider` — provider ID (currently only `"abacus"`)

## Key state behaviours

- `outputText` and `showDiff` are cleared on every `inputText` change (intentional UX).
- Error toast auto-dismisses after 6 seconds.
- Copy button label resets after 2 seconds.
- Button disabled by `!canPolish` (checks apiKey, provider, non-empty input, !isLoading), not by `isInteractionBlocked`.

## CSS

Plain CSS only — no Tailwind, no CSS-in-JS. Use the custom properties defined in `src/styles/global.css` (`--color-accent`, `--color-dark`, `--color-bg`, etc.).

## Package versions

Pin exact versions — no `^` or `~` in `package.json`. The existing `package.json` uses `^` throughout; do not extend this pattern.

## TypeScript

Strict mode (`astro/tsconfigs/strict`). Always use `{}` braces for `if` statements. Avoid type assertions; if unavoidable, add a comment explaining why it is safe.

## Adding a provider

Implement the `PolisherProvider` interface (`src/lib/providers/types.ts`), then register the new provider in `src/lib/providers/index.ts`. `SetupModal` automatically renders all entries from `AVAILABLE_PROVIDERS`.

## Generated files

`.astro/` is generated by `astro dev`/`astro build` and is gitignored. Do not edit it manually.

## `diff` package

`diff` (npm) is a **runtime** dependency used in the browser render path (`PolisherApp.tsx`). Keep it in `dependencies`, not `devDependencies`.

## Deployment

Vercel via GitHub integration (push to `main`). No CI workflow files in the repo.
